# Specs

This directory contains feature specifications that drive development.

## How It Works

1. Copy `_template.spec.md` and name it after the feature: `<feature-name>.spec.md`
2. Fill in the spec before writing any code
3. Share the spec with Claude — it will implement against it
4. Check off acceptance criteria as they are met

## Structure

```
specs/
  _template.spec.md       # Template for new specs
  decisions/              # Architecture Decision Records (ADRs)
  <feature>.spec.md       # One file per feature
```
